Playtest

AI playtest harness — module overview and configuration.

Description

The playtest module is the server side of an AI playtest harness: point an AI agent at this MUD, have it play through goals using standard tester "personalities," and get back a structured report. The agent connects on the dedicated AI port (Network.AI.Port), and everything this module does keys off that connection.

It does not create or provision any account — the agent logs into a character, or creates one via the normal new-player flow, exactly like a real player.

What it does (for any client on the AI port)

  • Per-round beacons — a Playtest.Round GMCP package each round carrying {round, hp, hp_max, sp, sp_max, room_id}, giving the agent a reliable pacing tick and a state snapshot to verify goals against. Requires the bundled gmcp module.
  • Structural safe mode — optional sandbox confinement: a tester that leaves a tagged zone is snapped back (fail-closed), keeping it away from live players.
  • Death protection — a high extra-lives count so a tester survives within the server's permadeath setting.
  • Admin commands — ai-flag / ai-list manage the engine's IsAI account flag (e.g. for leaderboard exclusion). Optional; the module's own behaviors don't depend on it.

The agent side — the mudagent adapter, tester personalities, goals, worked examples, and a reference driver — ships at GoMud-Module-Playtest-Harness.

Configuration

Set keys on the Config page (or in the module's data-overlays/config.yaml); changes take effect on the next server restart.

  • Enabled — master switch.
  • SafeMode + SandboxZoneTag — confinement (needs a tag to confine to).
  • DeathProtection — high extra-lives for testers.
  • Beacons — per-round Playtest.Round beacons (needs the gmcp module).

Room Tags

When SafeMode is on and SandboxZoneTag is set, the module reserves that tag and confines AI-port testers to rooms carrying it. The tag value is operator-chosen — there is no fixed default.
