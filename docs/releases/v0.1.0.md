# GoMud Playtest Harness v0.1.0

First public release of the AI playtest harness for [GoMud](https://github.com/GoMudEngine/GoMud):
point an AI agent at your MUD, have it play goals you define using standard
tester *personalities*, and get back a structured report.

## What's included

- **`playtest` module** (`playtest.tar.gz`) — server-side policy: provisions a
  flagged AI test account, applies a structural safe mode, and emits per-round
  `Playtest.Round` GMCP beacons for goal verification. Compiles into your GoMud
  server.
- **`mudagent` adapter** (`mudagent-<os>-<arch>` binaries) — a standalone telnet/
  GMCP client that logs in on the AI port and exposes a line-in / JSON-out
  protocol your agent drives. Download the one for your OS/arch.
- **Framework content** (`framework.tar.gz`, also in the repo) — the three tester
  personalities, the goals + report schemas, an engine-profile template, worked
  examples, and a reference Claude Code driver.

## Requirements

- **A GoMud build with the AI port** — merged into GoMud `master`
  (PRs #601/#602/#603), so any recent `master` has it. The AI port ships
  **disabled**; you enable it (`Network.AI.Port`).
- **The bundled `gmcp` module** (default in GoMud) for the per-round beacons.
- **Go** to compile the module into your server.

## Install

```bash
# from your GoMud checkout root:
go run . module install playtest          # fetches + sha256-verifies this release
go generate ./... && go build -o go-mud-server
```

Then enable the AI port and set the test-account password — see the
[Quick Start](https://github.com/GoMudEngine/GoMud-Module-Playtest-Harness#quick-start-operator)
and the [full usage guide](https://github.com/GoMudEngine/GoMud-Module-Playtest-Harness/blob/main/docs/usage/playtest-module.md)
(including the **config gotcha** and **troubleshooting**). New to writing goals?
Start with the [worked examples](https://github.com/GoMudEngine/GoMud-Module-Playtest-Harness/tree/main/framework/goals/examples).

## Notes

- **Beacon fix:** provisioning no longer leaves the test account registered as a
  live "phantom" session — that bug (found in pre-release validation) swallowed
  the per-round beacons and blocked the real client's login. Fixed; verified
  end-to-end on stock GoMud (clean login + live beacons).
- **Known follow-ups** (see [`docs/followups.md`](https://github.com/GoMudEngine/GoMud-Module-Playtest-Harness/blob/main/docs/followups.md)):
  the provisioned account starts as a pre-tutorial "ghost"; the admin-UI path for
  setting module config is still being finalized (edit the module's
  `files/data-overlays/config.yaml` for now).

## Trust & integrity

`module install` downloads **Go source** (not a binary) and you compile it
yourself; the archive's sha256 is pinned in the GoMud module registry. Read the
[security & trust model](https://github.com/GoMudEngine/GoMud-Module-Playtest-Harness/blob/main/docs/usage/playtest-module.md#8-security--trust-model)
before installing any module.
